## Что это за слой по архитектуре

**`AnimatedCellComponent` = Представительский UI-виджет (Presentation)**

Этот компонент отвечает за визуализацию изменения данных в ячейках таблиц. Вместо резкой смены текста он плавно анимирует переход.

---

## Контракт `AnimatedCellComponent`

### Ответственность

```text
– Получение текущего значения через @Input value
– Хранение "призрачного" (ghost) состояния старого значения
– Запуск CSS-анимаций для старого и нового значения при их смене
– Ручное уведомление ChangeDetectorRef о завершении анимаций
```

### Визуальные эффекты

#### 1. Ghost Animation (Улетающее старое значение)
- **Цвет**: `var(--accent-blue)`
- **Анимация**: `floatAway`. Старое значение плавно улетает вправо-вверх, увеличиваясь в масштабе до 1.3 и плавно исчезая.
- **Длительность**: 4.8 секунды.

#### 2. Pulse Animation (Пульсация нового значения)
- **Цвет**: `white` + `text-shadow`
- **Анимация**: `pulseNewValue`. Новое значение плавно увеличивается до 1.3 и уменьшается обратно.
- **Длительность**: 2.4 секунды за цикл (повторяется дважды за время отлета призрака).

## Пример в шаблоне

```html
<td>
  <app-animated-cell [value]="job.status"></app-animated-cell>
</td>
```

---

## Инварианты и правила

1. ✅ **Pure Presentation**: Не делает IO и не меняет состояние приложения.
2. ✅ **Handled Change Detection**: Использует `markForCheck()`, так как изменения в массиве `ghosts` происходят внутри асинхронного `setTimeout`.
3. ✅ **Pointer Events**: Улетающие значения имеют `pointer-events: none`, чтобы не мешать кликам по кнопкам в строке.
4. ✅ **Standalone**: Не имеет зависимостей от других сервисов.
