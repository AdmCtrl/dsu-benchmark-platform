### 📦 JobStore — контракт

**Назначение:**
Хранение и управление коллекцией `Job` в памяти приложения.

**Гарантии:**

* Единственный источник истины для списка jobs
* Все изменения jobs проходят только через методы `JobStore`
* `jobs$` всегда эмитит **новый массив**
* Объекты `Job` **могут мутироваться внутри стора**

**Не делает:**

* ❌ не принимает решений о статусах
* ❌ не знает про RUNNING / FINISHED логику
* ❌ не считает активные задачи
* ❌ не реагирует на события сервера

---

### 📘 Публичный API

```ts
jobs$: Observable<Job[]>
getJob(id): Job | undefined

add(job)
update(job)
remove(id)
setAll(jobs)
```

---

### ⚠️ Инварианты

* `update(job)` обновляет **существующий Job по id**
* Если job не найден — update **ничего не делает**
* Store **не валидирует** входные данные
* Ответственность за корректность `Job` лежит на фасаде
