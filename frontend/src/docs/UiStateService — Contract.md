## Что это за слой по архитектуре

**`UiStateService` = Глобальный UX-менеджер (Shared State)**

Этот сервис отвечает за то, чтобы пользователь всегда понимал, что происходит в приложении. Он управляет уведомлениями, прогресс-барами и глобальным флагом "занятости" (Busy).

---

## Контракт `UiStateService`

### Ответственность

```text
– Хранение и публикация списка активных уведомлений (messages$)
– Управление глобальным флагом isBusy$ (с защитой от мерцания)
– Предоставление универсального метода run() для обертки асинхронных задач
– Обновление прогресса выполнения задач (updateProgress)
```

### Публичный API

#### `isBusy$: Observable<boolean>`
Флаг, указывающий на наличие фоновых блокирующих процессов. Имеет встроенный `debounceTime(150ms)`, чтобы не мигать при мгновенных операциях.

#### `messages$: Observable<UiMessage[]>`
Список текущих алертов/уведомлений для отображения в `app-ui-notifications`.

#### `run<T>(label, observable, onContext?, options?): Observable<T>`
Основной метод для выполнения задач. 
- Автоматически создает уведомление с текстом `label`.
- Управляет инкрементом/декрементом `busyCount`.
- Обрабатывает ошибки, превращая их в "липкие" красные уведомления.
- По завершении скрывает уведомление через 3 секунды.

---

## Пример использования

```ts
this.uiState.run('Загрузка данных', this.api.fetchData()).subscribe();
```

---

## Инварианты и правила

1. ❌ **Никогда** не подписывается на SSE напрямую.
2. ❌ **Не хранит** доменные данные (только UI).
3. ✅ Используется как в **Services**, так и в **Components**.
4. ✅ Все изменения данных происходят внутри `NgZone`, гарантируя реактивность Angular.
