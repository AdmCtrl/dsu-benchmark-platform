## Роль `JobGridComponent` в архитектуре

**`JobGridComponent` = UI-компонент / orchestration layer**

Он:

* ❌ не хранит доменное состояние
* ❌ не подписывается на SSE
* ❌ не интерпретирует события
* ❌ не считает running / totals
* ✅ отображает данные из Store
* ✅ дергает Facade / Services по user-action
* ✅ управляет чисто UI-состоянием (openLogIds)
* **🆕 Реализует Anti-jitter верстку** (фиксированные колонки, резерв скролла)
* **🆕 Использует `AnimatedCellComponent`** для визуализации обновлений данных
---

## Контракт `JobGridComponent`

### Ответственность

```text
– Отображает список Job (панель `.job-container`)
– Показывает derived UI-метрики (running count)
– Делегирует все доменные действия фасаду / сервисам
– Управляет локальным UI state (раскрытые логи)
– Управляет жизненным циклом SSE (init/destroy фасада)
– **🆙 Контракт верстки**: 
    – `table-layout: fixed` (статичные колонки)
    – `border-collapse: collapse` (монолитные строки)
    – `sticky` header (шапка таблицы всегда сверху)
    – Внутренний скролл только внутри `.table-scroll-wrapper`
```
